# 宿note 周辺ガイドLP 要件定義書

## 0. プロジェクト概要
- 目的: 宿の“推しスポット”を厳選し、ゲストの迷いを減らして満足度とレビューを向上させ、オーナーの問い合わせ対応負荷を軽減する。
- 提供物: 1ページLP（日本語。オプションで2言語切替 or 日英併記）、入稿データ一式、更新手順書。
- 制作方針: 大きく作らず、正しく保つ。静的・軽量構成とし、更新のしやすさを最優先する。

## 1. KPI / 成果目標
- 滞在前〜滞在中の周辺案内に関する質問を20%削減（オーナー申告ベース）。
- 主要スポットカードの地図クリック率15%以上（公開後3週間平均）。
- Lighthouseモバイル計測で Performance 90以上 / SEO 90以上を達成。

## 2. ターゲット
- 一次ターゲット: 当該宿のゲスト（国内外）。
- 二次ターゲット: 宿オーナーおよび運用担当スタッフ。

## 3. 情報設計 / ページ構成
- Hero: サービス名、タグライン、「地図で見る」「LINE/予約」CTA。
- カテゴリフィルタ: 朝食 / ランチ / カフェ / ディナー / 子連れ / 雨OK / 近距離(車10分)。
- スポット一覧（カード）: 画像、店名、一言コメント（20–30字）、営業時間、定休日、価格帯、所要時間（車/徒歩）、ボタン（Google Map / 公式 / 予約）。
- モデルコース（半日×1）: 朝 → 昼 → カフェ → 夕。スポットID参照で表示。
- 地図エンベッド: 主要スポットのピンと凡例を表示するGoogle Maps埋め込み。
- Q&A: 混雑時間・支払い・駐車など5〜7項目のFAQアコーディオン。
- 料金・プラン紹介: 通常/期間限定プラン、2言語切替および併記オプション説明。
- 制作の流れ: ヒアリング → 初稿 → 修正 → 公開 → 更新。
- 実績・お客様の声（仮掲載可）。
- プロフィール（制作者紹介）。
- お問い合わせ / CTA: メール、LINE、見積依頼。
- フッター: 連絡先、対応時間、著作権/権利表記。

## 4. コンテンツ要件
- タグライン: 「大きく作らず、正しく保つ。」（40字以内）。
- 特徴: 3点、各40〜60字。
- 一言コメント: 各スポット20〜30字。ゲストに効く具体表現を用いる。
- モデルコース説明: 各ステップ30〜50字。
- プロフィール: 120〜180字。
- Q&A: 質問20〜30字 / 回答60〜100字。

## 5. データモデル / 入稿仕様
- 運用形式: CSVまたはGoogleスプレッドシート（推奨）。ビルド不要で差し替えまたは自動反映可能とする。
- データテンプレート: `spots.csv`

| カラム名 | 説明 |
| --- | --- |
| name | 店名 |
| category | 分類。値は `breakfast` `lunch` `cafe` `dinner` `kids` `rain` `near10` |
| address | 住所 |
| maps_url | Google Mapリンク |
| web_url | 公式サイトリンク |
| reserve_url | 予約サイトリンク |
| hours | 営業時間 |
| closed | 定休日 |
| price_range | 価格帯 |
| payment | 支払い方法 |
| parking | 駐車情報 |
| kid_friendly | 子連れ可否 |
| host_note | 20〜30字のコメント（例: 朝9時前は並ばず入れる） |
| time_by_car_min | 車での所要時間（分） |
| time_on_foot_min | 徒歩での所要時間（分） |
| image_file | 画像ファイル名 |

- 画像仕様: 横1280px以上、JPG/WEBP、1枚あたり500KB以下、権利クリア済。
- 多言語設定: 切替型（/ と /en/ を分離し hreflang 設定）と併記型を選択可能。

## 6. デザイン要件
- トーン: クリーンで温度感のあるローカル感。白ベース＋アクセント1色。
- タイポグラフィ: 見出しはセミボールドのサンセリフ。本文は可読性重視のサンセリフ（モバイル14〜16px相当）。
- UIパターン: 角丸16〜20pxのカードと柔らかい影、トグル式フィルタ、固定フッターCTA（TEL・LINE・MAP）。
- アイコン: カテゴリ・設備・支払い情報にピクトを使用。
- モーション: フェードや小さなトランジション（200〜300ms）に限定。
- レスポンシブ: モバイル基準（360px〜）→ タブレット（768px〜）→ デスクトップ（1024px〜）。
- アクセシビリティ: コントラスト比WCAG AA準拠、タップ目標48px、画像はalt必須、キーボード操作対応。

## 7. 機能要件
- フィルタ: カテゴリ多選択・即時絞り込み。
- 並び順: デフォルトで近距離優先、次いで人気順（任意）。
- 地図: Google Maps埋め込み（カスタムピン）。各カードから外部マップへ遷移。
- モデルコース: スポットIDを参照しルート表示。リンクでマップ起動。
- 多言語対応: 切替 or 併記を設定フラグで制御。
- SEO / OGP: title 50字前後、meta description 90〜120字、og:image 1200×630。
- 解析（任意）: MAP・外部リンクのクリックイベントを計測。

## 8. 技術要件 / 環境
- 配信: 静的ホスティング（標準: GitHub Pages、任意でお客様サーバー）。
- ビルド: 不要。プレーンHTML/CSS/最小限のJSで構築し、手修正可能とする。
- 更新ワークフロー: 代行更新は月1回最大10件・1営業日以内反映。セルフ更新オプションはGoogleシート公開→CSV読み込み or GitHubでCSV編集。
- ドメイン: お客様名義推奨。CNAME設定とHTTPS対応が必要。

## 9. 料金表表示
- 通常（1言語）: 初期費用 50,000円（税別） / 期間限定 30,000円。
- 2言語切替: 通常 80,000円 / 期間限定 50,000円（SEO・可読性に強い）。
- 日英併記: 通常 65,000円 / 期間限定 60,000円（低コスト・SEO弱）。
- メンテナンス: 月次 5,000円（最大10件、同日一括）。
- 年1回更新: 10,000円（最大10件）。
- 都度/緊急対応: 5,000円/回（当日〜翌営業日）。
- 導入支援: +30,000円（ChatGPT活用レクチャーとマニュアル含む）。

## 10. 制作フロー / スケジュール
1. ヒアリングおよび素材受領（CSV・画像・導線情報）。
2. 初稿を5営業日で共有。
3. 修正2往復。
4. 公開（目安10営業日）。
5. 運用: 月1回更新（最大10件 / 1営業日以内反映）。

## 11. 品質基準 / 受入テスト
- 表示速度: LighthouseモバイルPerformance 90以上。
- 画像: すべて `loading="lazy"`、width/height指定必須。
- リンク: 外部・地図・予約リンクで404ゼロを確認。
- アクセシビリティ: キーボード操作、alt属性、コントラストAAを確認。
- 文言: 数字・表記ゆれを統一し、誤字脱字ゼロ。
- 多言語（該当時）: hreflangと言語切替の反映を確認。
- SEO: title・description設定、OGP表示確認。

## 12. 権利・法務
- 写真: 商用利用可素材のみ使用。店舗提供が得られない場合はテキストのみで掲載。
- 著作権: コードは製作者帰属。サイト利用権はお客様へ非独占的に許諾。データ（CSV/画像/文言）はお客様に帰属。
- キャンセル: 着手後は実費相当（目安50%）を請求。
- 表記: フッターに © 宿note / kzcodelab、問い合わせ先、最終更新日を記載。

## 13. 画面コンポーネント仕様
- Header: ロゴ/サービス名、ハンバーガーメニュー、言語切替（オプション）。
- FilterBar: Pillボタン（多選択可）。
- SpotCard: 16:9画像、店名、一言コメント、基本情報アイコン列、所要時間（車/徒歩）、地図/予約ボタン。
- CourseBlock: 時刻＋スポット名のタイムライン表現。
- FAQ: アコーディオン形式。
- CTA: 固定フッター（TEL・LINE・MAP）とページ末ボタン。

## 14. コピー骨子
- 見出し: 「推しが価値。迷わない旅導線。」
- 本文: 宿主が胸を張ってすすめる店だけを厳選。所要時間と一言を添えて、初めてでも迷わない体験を提供する。
- CTA: 「まずは推しを5件、教えてください」（3〜5営業日で初稿）。

## 15. リスクと対策
- 情報鮮度劣化: 月1回更新契約を必須化。閉店・時短は緊急枠で優先反映。
- 写真不足: スマホ撮影＋簡易補正の案内を提供。店舗提供依頼テンプレートを用意。
- 運用ミス: CSVテンプレートで入力チェックとカテゴリ制限を行う。

## 16. 受入条件
- KPI計測設定済（クリックイベント等の代替指標含む）。
- 仕様どおりのUI/文言/料金表/連絡先がLPに反映されている。
- 更新手順書（1枚）とデータ一式を納品済み。
- スポット15件、モデルコース1本、OGP設定が完了している。
